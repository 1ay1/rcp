Modern agent architectures have three interoperability seams. RCP fills the one
that was missing.

<div class="rcp-stack not-content">
  <div class="layer acp"><b>ACP</b><span>editor ⇄ agent</span></div>
  <div class="layer mcp"><b>MCP</b><span>agent ⇄ tools</span></div>
  <div class="layer rcp"><b>RCP</b><span>agent ⇄ retrieval</span></div>
</div>

```
  client  ⇄  ACP  ⇄  agent  ⇄  MCP  ⇄  tools
                       │
                       └──────  RCP  ⇄  retrieval engine(s)
```

- **[ACP](https://agentclientprotocol.com)** — the Agent Client Protocol. Lets a
  code editor / IDE drive a coding agent. *(editor ⇄ agent)*
- **[MCP](https://modelcontextprotocol.io)** — the Model Context Protocol. Lets
  an agent call tools and resources. *(agent ⇄ tools)*
- **RCP** — the Retrieval Context Protocol. Lets any client pull grounding
  context from any RAG engine. *(agent ⇄ retrieval)*

All three share JSON-RPC framing, an `initialize`/capabilities handshake, `_meta`
extensibility, and cursor-based pagination. Learn one and you are immediately
productive in the others.
